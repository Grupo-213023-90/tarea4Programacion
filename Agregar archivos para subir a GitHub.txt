Agregar archivos para subir a GitHub

git add .
git status

Si solo se quiere agregar un archivo especifico

git add index.html
git config --global user.name "Fredy Andres"
git config --global user.email "tangarifefredy1995@gmail.com"

Conectar tu proyecto con GitHub
git remote add origin https://github.com/Grupo-213023-90/tarea4Programacion.git

Esto le dice a Git: "el repositorio remoto (en GitHub) se llama origin y está en esta URL". Es como darle a Git la dirección de destino. Solo se hace una vez.

Hacer el primer commit (guardar el estado)
git commit -m "primer commit"

Este sirve para guardar el estado de tu proyecto en ese momento. El mensaje entre comillas describe qué hiciste. Git guarda ese historial para siempre y se puede volver a cualquier punto.

git branch

para ver en que rama estamos

git branch -M main
cambiarnos a rama principal main

Subir tu código a GitHub
git push -u origin main

git pull origin main --allow-unrelated-histories ## si sale error

git pull origin main –rebase “si sale error al cargar”

push -> sube tus commits al servidor

origin -> el nombre que le diste a GitHub en el paso anterior

main -> la rama que estás subiendo


De ahí en adelante el ciclo es siempre este:

Haces cambios en tus archivos

Los agregas

git add .

Los confirmas con un mensaje

git commit -m "descripción de lo que cambiaste"

Los subes

git push
